Impact Demo

Preamble

The Impact Demo project is setup so that there is a natural progression of functionality being added as students move from step 1 to step 9. The user will modify the src/com/ibm/mysurance/Mysurance.java file to point to a new HTML file that corresponds to the next step in the project.

Step 1

In the first step the student will run the project from Eclipse so they can get used to launching the Android emulator. The "step1.html" file is very simple. It includes a referece to the phonegap.0.9.4.js file but it is worth noting that PhoneGap is not ready to be used until the "deviceready" event is fired. Take note of the "onBodyLoad()" method which in turn registers an evnet listenter for the "deviceready" event. When the event is fired the "onDeviceReady()" method is run and PhoneGap is ready to be used. Students will be able to see that the event has been fired by looking at the console logs. The string "onDeviceReady()" will be printed to the console. Users will be able to view the console by running "adb logcat" from a Terminal.

Step 2

In step 2 we begin to introduce features from Mobile Dojo including views, headings and transistions.

Step 3

In step 3 we can transistion to our first sub-page. The settings page is only provided for people who want to debug the applicaiton on their own machines and will not be used in the class.

Step 4

In step 4 new functionality is introduced in the form of SQL Lite DB. The users profile is read from and stored to a SQL Lite DB to keep it persistent across runs of the applicaiton. Additionaly, students will be able to test the interaction with Web Sphere and a JAX-RS service. By using an XmlHttpRequest the application sends the users profile information to the server in order to be saved. The first update will create a new user ID, the second update will modify the exiting information on the server.

Step 5

Step 5 is pretty much the same as step 4 except this time we are using vehicle information.

Step 6

In step 6 we introduce a new pane for collecting accident information. The first two entries on the page, "Call the Police" and "Call the Tow Truck" will fire off a map application to give users an idea of where the closest Police or Tow Truck can be found.

Step 7

In step 7 the student can record the information of the other driver in the accident.

Step 8

In step 8 the student can record the location of where the accident took place. In order to make data entry easier the user can request the device use the current location. In the first step the device retrieves the latitude and longitude from the built in GPS. It then users that data to query a web service for the street, city, state, etc. information.

Step 9

In step 9 the student can use the camera functionality to take a picture. Sadly you are stuck with the default Android picture. Then the student can upload the photo to the JAX-RS service.
